### 1. 개요
앱을 열면 사용자가 가장 처음에 마주할 화면이다.
### 2. 관계
1. 사용자
### 3. 기본 흐름
1. DB 에서 프로젝트를 가져온다.
2. 가장 첫번째 순서의 프로젝트를 메인화면에 출력한다..
3. 프로젝트의 섹션, 작업이 정렬하여 메인화면에 출력한다.
4. 왼쪽 탭에서 아래의 작업으로 넘어갈 수 있다.
    - [프로젝트 생성](#프로젝트-생성---uc-p001)
    - 즐겨찾기
5. 프로젝트 화면에서 아래의 작업으로 넘어갈 수 있다.
    - [섹션 생성](#섹션-생성---uc-s001)
    - [작업 생성](#작업-생성---uc-t001)
### 4. 대안 흐름
### 5. 예외 흐름
#### E1. 아무 데이터가 없는 경우
1. 화면 중앙에 프로젝트 생성 버튼이 표시된다.
2. 프로젝트 생성 버튼을 누를 경우 [프로젝트 생성 시나리오](#프로젝트-생성---uc-p001)의 기본 흐름 2부터 시작한다.

## 프로젝트 생성 - UC-P001
### 1. 개요
사용자는 프로젝트를 등록한다.
### 2. 관계
1. 사용자
### 3. 기본 흐름
1. 메인 화면 왼쪽탭의 해당 프로젝트의 설정 버튼을 눌러 '프로젝트 생성' 혹은 메인화면의 '프로젝트 생성' 버튼을 클릭한다.
2. 프로젝트 생성 다이얼로그를 출력한다.
3. 프로젝트 이름을 입력한다.
4. 색상을 선택한다.
5. 즐겨찾기 여부를 선택한다.
6. 보기 방식을 선택한다.
7. '저장'을 선택한다.
8. 저장될 때까지 로딩을 보여준다.
9. 프로젝트 생성이 완료되어 로딩을 끝내고 생성되었음을 나타내는 메세지를 출력한다.
11. 다이얼로그를 내린다.
12. 메인화면을 갱신한다.
### 4. 대안 흐름
#### E1. 모바일 화면 기본 흐름
1. 드로어를 열어 프로젝트 탭의 프로젝트 설정 버튼을 눌러 '프로젝트 추가' 버튼을 클릭한다.
2. 프로젝트 생성 화면으로 넘어간다. 이후 동일
11. 메인화면으로 이동한다.
### 5. 예외 흐름
#### E1. '취소'를 선택한 경우
1. 프로젝트 생성 다이얼로그를 내린다.
#### E2. 모바일 화면에서 '취소'를 선택한 경우
1. 메인화면으로 넘어간다.
#### E3. 저장이 안될경우
1. 프로젝트 저장이 3초 이후에도 안될 경우 '프로젝트 저장에 실패했습니다.' 메세지를 출력한다.
2. 예외흐름 E1, E2를 진행한다.

## 섹션 생성 - UC-S001
### 1. 개요
사용자는 섹션을 생성한다.
### 2. 관계
1. 사용자
### 3. 기본 흐름
1. 프로젝트 화면에서 '섹션 추가' 버튼을 누른다.
2. '섹션 추가' 버튼이 이름을 입력하고 버튼이 있는 UI로 바뀐다.
3. 섹션의 이름을 입력한다.
4. '추가'와 '선택' 중 '추가'를 선택한다.
5. 섹션을 생성하는 UI가 사라지고 '섹션 추가' 버튼 상태로 돌아온다.
6. 화면이 갱신되어 추가한 섹션이 나타난다.
### 4. 대안 흐름
#### E1. 모바일 화면 기본 흐름
2. 섹션 생성 다이얼로그가 나타나는 것으로 변경한다.
5. 색션 생성 다이얼로그가 내려간다.
### 5. 예외 흐름
#### E1. '취소'를 선택한경우
1. 모바일인 경우 섹션 생성 다이얼로그가 내려간다.
2. 데스크탑일 경우 섹션 추가 상태로 바뀐다.
#### E2. 생성이 안될경우
1. 섹션 추가 후 3초가 지나도 생성이 안되면 '생성을 실패했습니다' 메세지를 출력한다.

## 작업 생성 - UC-T001

1. 사용자
### 3. 기본 흐름

4. 설명을 입력한다.
5. 마감 날짜를 설정한다.
6. 우선순위를 설정한다.
7. 라벨을 설정한다.
8. 미리 알림을 설정한다.
9. '작업 추가'버튼과 '취소' 버튼 중 '작업 추가' 버튼을 누른다.
10. 작업 생성 UI가 작업 추가 버튼으로 돌아간다.
11. 작업이 생성되어 UI가 갱신된다.
### 4. 대안 흐름
#### E1. 모바일 화면
2. 다이얼로그가 나타난다.
10. 다이얼로그가 내려간다.
#### E2. 기본 흐름 선택사항
4. 설명이 없으면 비운다.
5. 마감 날짜가 없으면 비운다.
6. 우선순위가 없으면 가장 낮음으로 설정한다.
7. 라벨이 없으면 비운다.
8. 미리 알림이 없으면 비운다.
### 5. 예외 흐름
#### E1. 필수 내용 없음
1. 작업 이름이 없으면 '작업 추가' 버튼은 비활성화 된다.
#### E2. 생성 실패
1. 생성 실패시 '저장에 실패했습니다.' 메세지를 출력한다.

## 작업 이동 - UC-T005
### 1. 개요
사용자는 작업 순서를 변경한다.
### 2. 관계
사용자
### 3. 기본 흐름
1. 작업 카드 누른다.
2. 작업 카드 드래그 가능한 상태가 된다.
3. 작업 리스트의 가장자리 혹은 작업 사이에 둔다.
4. 작업 카드를 둔곳으로 이동시킨다.
5. 이동이 완료되어 UI가 갱신된다.
### 4. 대안 흐름
1. 모바일 화면에서는 길게 누른다.
### 5. 예외 흐름
#### E1. 이동 취소
1. 제자리에 다시 두면 이동이 취소된다.
2. 작업 리스트 스크롤의 수직 방향 가장자리에 놓으면 이동이 최소된다.
#### E2. 이동 실패
1. '변경에 실패했습니다' 메세지를 출력한다.
