# 유즈케이스 시나리오
## 앱 실행 - UC-M001
### 1. 개요
앱을 열면 사용자가 가장 처음에 마주할 화면이다.
### 2. 관계
1. 사용자
### 3. 기본 흐름
1. DB 에서 프로젝트를 가져온다.
2. 가장 첫번째 순서의 프로젝트를 메인화면에 출력한다..
3. 프로젝트의 작업 종류, 작업이 정렬하여 메인화면에 출력한다.
4. 왼쪽 탭에서 아래의 작업으로 넘어갈 수 있다.
    - [프로젝트 생성](#프로젝트-생성---uc-p001)
    - 즐겨찾기
### 4. 대안 흐름
### 5. 예외 흐름
#### E1. 아무 데이터가 없는 경우
1. 화면 중앙에 프로젝트 생성 버튼이 표시된다.
2. 프로젝트 생성 버튼을 누를 경우 [프로젝트 생성 시나리오](#프로젝트-생성---uc-p001)의 기본 흐름 2부터 시작한다.

## 프로젝트 생성 - UC-P001
### 1. 개요
사용자는 프로젝트를 등록한다.
### 2. 관계
1. 사용자
### 3. 기본 흐름
1. 메인 화면 왼쪽탭의 해당 프로젝트의 설정 버튼을 눌러 '프로젝트 생성' 혹은 메인화면의 '프로젝트 생성' 버튼을 클릭한다.
2. 프로젝트 생성 다이얼로그를 출력한다.
3. 프로젝트 이름을 입력한다.
4. 색상을 선택한다.
5. 즐겨찾기 여부를 선택한다.
6. 보기 방식을 선택한다.
7. '저장'을 선택한다.
8. 저장될 때까지 로딩을 보여준다.
9. 프로젝트 생성이 완료되어 로딩을 끝내고 생성되었음을 나타내는 메세지를 출력한다.
11. 다이얼로그를 내린다.
12. 메인화면을 갱신한다.
### 4. 대안 흐름
#### E1. 모바일 화면 기본 흐름
1. 드로어를 열어 프로젝트 탭의 프로젝트 설정 버튼을 눌러 '프로젝트 추가' 버튼을 클릭한다.
2. 프로젝트 생성 화면으로 넘어간다. 이후 동일
11. 메인화면으로 이동한다.
### 5. 예외 흐름
#### E1. '취소'를 선택한 경우
1. 프로젝트 생성 다이얼로그를 내린다.
#### E2. 모바일 화면에서 '취소'를 선택한 경우
1. 메인화면으로 넘어간다.
